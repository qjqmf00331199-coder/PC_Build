# [디자인 가이드] 트라이핏 (TriFit)

## 1. 톤앤매너
- **다크 테크 (Linear/Vercel 스타일)**: 배경 거의 블랙에 가까운 다크그레이, 카드/구분선 미세한 대비, 포인트 컬러는 절제해서 소량만 사용.
- 정보 밀도 높은 도구형 UI. 장식보다 가독성/스캔성 우선.

## 2. 컬러 팔레트

| 용도 | 색상 | 값 |
|---|---|---|
| 배경(base) | 거의 검정 | `#0A0A0B` |
| 카드/패널 배경 | 다크그레이 | `#151517` |
| 구분선/보더 | 저채도 그레이 | `#27272A` |
| 본문 텍스트 | 오프화이트 | `#E4E4E7` |
| 보조 텍스트 | 그레이 | `#9CA3AF` |
| 포인트(강조/CTA) | 네온 라임 | `#B6F13B` |
| 호환 성공 | 그린 | `#22C55E` |
| 호환 불가(경고) | 레드 | `#EF4444` |
| 비권장(주의) | 앰버 | `#F59E0B` |

## 3. 타이포그래피
- **폰트**: Inter (영문/숫자용) + Pretendard (한글용) 조합 — Linear류 다크테크 톤에 잘 맞음.
- 스펙 수치(전력, 클럭 등)는 `font-mono` (예: JetBrains Mono) 적용 — 데이터 느낌 강조.

## 4. 컴포넌트 스타일 (Tailwind + shadcn/ui)

- **카드**: `bg-[#151517] border border-[#27272A] rounded-lg` — 과한 라운딩/그림자 지양 (다크테크는 flat에 가까움).
- **호환성 배지**: 성공/경고/불가 3단계 색상 배지, 아이콘 포함 (lucide-react: CheckCircle2 / AlertTriangle / XCircle).
- **버튼**: 기본 상태 `bg-[var(--accent)]` + 검정 텍스트(대비 확보), hover 시 `opacity-90` (scale 애니메이션은 최소화 — 다크테크는 절제된 인터랙션).
- **선택 카드(부품)**: 선택 시 `border-[var(--accent)]` 테두리로 표시, 배경색 크게 안 바뀜.
- **배경**: 32px 그리드 라인(저채도) + 우상단 accent 라디얼 글로우로 다크테크 무드 강조.

## 5. 레이아웃 (Apple 구매 화면 스타일)
- **데스크탑 우선**. `max-w-6xl` 컨테이너, 2컬럼 구조.
- **왼쪽 패널 (sticky, 스크롤 고정)**: PC 케이스 내부 단면 일러스트 SVG.
  - 케이스 외곽 + 메인보드 + CPU/쿨러 + RAM 2개 + GPU + SSD + PSU + (케이스 자체) 각 부품을 개별 도형으로 배치.
  - 기본 상태: 회색(`--fill-control`, 미선택).
  - 부품 선택 즉시 해당 도형 색 전환 애니메이션(`transition: fill 0.3s ease`):
    - 초록(`--fill-success`) = 호환
    - 노랑(`--fill-warning`) = 주의(작동은 하나 성능 제한 등)
    - 빨강(`--fill-danger`) = 호환 불가
  - **호환 불가 상태에서도 선택 자체는 차단하지 않음** — 경고 색상 + 하단 텍스트 메시지로만 안내 (즉시 트리거, 경고만).
  - 일러스트 하단에 상태 텍스트 1줄 + 색상 범례(초록/노랑/빨강) 고정 표시.
- **오른쪽 패널**: 카테고리별 카드 세로 나열(CPU→메인보드→RAM→SSD→GPU→PSU→케이스→쿨러), 각 카드 상단에 진행 표시(`N / 8`), 카드 내부에 선택 가능한 부품 옵션 버튼.
- 모바일: 왼쪽 일러스트를 상단 고정 바 형태의 축소 버전으로 전환 (스크롤 시 오른쪽 카드 목록만 스크롤).

## 6. 라이브러리
- **shadcn/ui**: Tailwind 기반, 다크테크 톤 커스터마이징 쉬움.
- **lucide-react**: 아이콘 (호환성 배지, 카테고리 아이콘).
- 애니메이션: 순수 CSS transition (`transition-colors duration-150`) — Framer Motion 불필요, 도구형 UI는 반응속도가 더 중요.
