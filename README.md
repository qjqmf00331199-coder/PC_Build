## 왜 만들었나

### 1. 문제 정의

美 소비자제품안전위원회(CPSC)는 2024년 2월, GPU 전원 커넥터용 각도 어댑터 약 25,300개를 화재·화상 위험으로 정식 리콜했습니다 — 규격 전환기에 사용자가 스스로 호환성·안전성을 판단하기 어렵다는 문제가 커뮤니티 루머가 아니라 정부기관 공식 문서로 확인된 사례입니다. PC 조립 시 **소켓/전원 규격이 자주 바뀌어** 소비자가 호환성을 스스로 판단하기 어렵고, 오판 시 **화재·성능 저하** 등 실질적 리스크로 이어진다는 데서 출발했습니다.

### 2. 객관적 근거

**기술 출처**

- **[美 CPSC 공식 리콜](https://www.cpsc.gov/Recalls/2024/GPU-Angled-Adapters-Recalled-Due-to-Fire-and-Burn-Hazards-Manufactured-by-CableMod) (근거 등급 최상)** — 2024년 2월, CableMod의 12VHPWR 앵글 어댑터(V1.0/V1.1) 약 25,300개가 화재·화상 위험으로 리콜됐습니다. 커넥터 말단이 헐거워지며 과열되어 GPU에 융착되는 현상이 272건 보고됐고, 미국 내 재산 피해 신고액이 7만 4,500달러를 넘었습니다. "화재 위험"이 커뮤니티 리포트 수준이 아니라 정부기관 리콜 문서로 입증된, 근거 신뢰도가 가장 높은 사례입니다.
- **[Intel 공식 지원 문서](https://www.intel.com/content/www/us/en/support/articles/000099723/processors.html)** — LGA1851 소켓은 LGA1700과 물리적 크기·치수가 동일하지만 하위 호환되지 않으며, 800시리즈 칩셋 메인보드가 새로 필요하고 기존 700/600시리즈 칩셋과는 호환되지 않는다고 명시합니다. 단, 쿨러 장착용 마운트홀 위치는 동일해 기존 LGA1700용 쿨러는 그대로 사용 가능합니다. "크기가 같으면 호환된다"는 소비자 직관이 틀릴 수 있음을 제조사 1차 문서로 보여주는 사례입니다.
- **[Tom's Hardware 기술 분석](https://www.tomshardware.com/pc-components/gpus/melting-power-connectors-a-comprehensive-timeline-of-the-rtx-4090s-biggest-flaw)** — PCI-SIG가 초기 12VHPWR의 문제 제기 이후 12V-2x6로 규격을 개정했지만, 개별 핀이 흘려야 하는 전류에 비해 안전마진이 여전히 낮다고 분석합니다. 반대로 기존 6/8핀 PCIe 커넥터는 안전마진이 훨씬 커 과부하 손상 사례가 거의 없었다는 점과 대조됩니다 — 규격이 한 번 바뀐 뒤에도 구조적 위험이 완전히 해소되지는 않았다는 근거입니다.
- **ATX 3.0 / PCIe CEM 5.0 트랜지언트 부하 스펙** — ATX 3.0에 포함된 PCIe CEM 5.0 ECN(엔지니어링 체인지 노트)은 PSU가 정격 지속전력의 최대 3배에 달하는 100마이크로초 순간 부하(transient spike)를 처리할 수 있어야 한다고 규정합니다. 단순히 GPU TDP와 PSU 정격 용량만 숫자로 맞추는 방식이 왜 위험한지를 스펙 문서 수준에서 뒷받침합니다.
- **[Corsair 공식 지원 문서](https://help.corsair.com/hc/en-us/articles/32465644341265)** — RTX 5090(소비전력 575W)에는 1,000W, RTX 5080(소비전력 360W)에는 850W PSU를 권장합니다. TDP만 보고 PSU 용량을 산정하면 과소·과대 산정 모두로 이어질 수 있다는 점을 제조사 권장치로 재확인합니다.
- **IT동아 뉴스줌인(2024.01.29)** — ATX 3.0 → 3.1 전환이 1~2년 만에 이뤄졌고, 그 사이 GPU 전원 커넥터도 12VHPWR → 12V-2x6로 세대가 교체됐습니다. 소비자가 따라가기 버거운 속도로 규격이 바뀌고 있다는 배경입니다.

**자체 설문 (N=31, PC 조립 경험자 N=18)**

> **핵심 인사이트**: 기술 문서로 확인된 실제 리스크(소켓 전환, 전원 규격 변경, 파워 용량 오판)와 설문으로 확인된 사용자 행태(경험자조차 88.9%가 초보·중수, 결정까지 1주 이상 걸리는 비율 50%)가 겹치면서, 이 문제는 "정보를 더 보여주는 것"이 아니라 "판단을 대행해주는 것"으로 풀어야 한다고 결론 내렸습니다.

### 3. 해결 방안 (핵심 기능)

전원 규격까지 포함한 **실시간 호환성 체크** 기능 하나에 집중해, 사용자의 판단을 대행하는 가장 작은 단위부터 시작했습니다.

**3-1) 실시간 자동 호환성 체크**

- 부품 선택 시마다 소켓·전원·크기 호환성 즉시 확인
- 사이드바에 선택 목록 + 예상 총액 동시 표시
- 즉시 연산 항목 예: CPU-메인보드 소켓 일치, PSU 전력 총합/여유율, 케이스-라디에이터 간섭
- 실사용 검증 예: i5 14600K 선택 → LGA1700 즉시 표시 / GPU 추가 시 소비전력 200W → 420W 재계산 / PSU 여유율 35.4% 자동 산출

**3-2) AI 추천 기능**

입력은 용도 → 우선순위 → 브랜드 → 부가요구 → 자유서술 5단계, 사용자 입력을 구조화된 프롬프트로 변환해 AI가 DB 내 실존 조합을 반환하면 호환성 엔진으로 다시 검증한 뒤에만 제안합니다.

AI 모델은 배포 이후 다음 순서로 교체했습니다.

1. **NVIDIA NIM (Llama 3.1 70B) → 폐기.** 실서비스에서 AI 추천이 거의 항상 실패해 "검증된 기본 조합"으로만 대체됐습니다. 원인 두 가지: (1) 네트워크/타임아웃 에러가 재시도 루프 밖에서 던져져 3회 재시도가 사실상 작동하지 않음, (2) 부품 DB가 332개로 커지면서 프롬프트가 비대해짐.
2. **Groq (Llama 3.3 70B Versatile)로 교체.** 재시도 버그를 고치고 응답 속도도 개선했지만, 무료 티어 TPM(분당 토큰) 한도가 12,000인데 프롬프트가 13,458토큰이 필요해 매 요청 100% 실패하는 근본적 한계를 발견했습니다. 재시도해도 한도 자체를 못 넘으니 의미가 없었습니다.
3. **Gemini 2.5 Flash로 재교체.** 무료 티어 토큰 여유가 훨씬 크고, 응답의 `cachedContentTokenCount` 필드로 확인한 결과 반복 프롬프트의 95%가 자동으로 캐시 히트되어 별도 구현 없이 얻는 이득이 컸습니다. 여러 명이 동시에 시연해도 병목이 가장 덜했습니다.
4. **Gemini 2.5 Flash → 무료 티어 일일 한도로 재차 실패, Groq로 최종 확정.** Gemini는 응답 속도·캐시 히트율은 좋았지만 실서비스 트래픽에서 일일 요청 한도(무료 티어 20건/day)에 바로 걸렸습니다. 429 응답의 `retry-after` 값이 요청마다 9.7s → 36.5s → 59.6s로 계속 늘어나는 걸 보고, 이게 분당 한도가 아니라 자정까지 리셋되지 않는 일일 한도라는 걸 확인했습니다. 즉 대기 로직을 아무리 정교하게 짜도 성공률은 오르지 않고 응답 시간만 늘어나는 구조라, 대기 로직을 걷어내고 429가 뜨면 즉시 안전 조합으로 대체하도록 바꿨습니다.
5. **폴백 발생 시 UX 보강.** 재시도해도 해결되지 않는 실패는 즉시 폴백 처리하되, "AI가 조합을 못 찾음" 같은 오해성 문구 대신 "요청 한도 초과, 잠시 후 다시 시도해주세요"처럼 사유를 명확히 구분해 보여주도록 했습니다. 폴백 배너에 재시도 버튼을 추가해, 한도가 풀린 뒤 사용자가 다시 시도하면 실제 AI 추천을 받을 수 있게 했습니다.

> **교훈**: 무료 티어 API를 프로덕션에 붙일 때는 "느려서 실패하는가"와 "한도에 걸려서 실패하는가"를 반드시 구분해야 합니다. 전자는 재시도/타임아웃 튜닝으로 풀리지만, 후자는 재시도가 오히려 응답 시간만 늘리는 역효과를 냅니다. 이 케이스는 처음엔 "타임아웃"으로 오진단됐다가, 실제 에러 바디(429 RESOURCE_EXHAUSTED, quota 메시지)를 로그로 직접 확인하고서야 정확한 원인(요청 한도)을 잡을 수 있었습니다.

**3-3) 성능 병목 진단**

CPU·GPU 선택 시 "병목 발생 가능" 배지를 표시하고, 티어·점수 산출과 격차 설명 문구를 자동 생성합니다. 룰 기반 매칭으로 구현해 Phase 1과 함께 배포했습니다.

### 4. 제작 회고

문제 정의부터 출시까지 사용자 편의 기능을 찾아 적용하고 버그를 수정했습니다. 출시 이후에는 비즈니스 모델을 구상하며 법적 리스크(개인정보처리방침 부재, 이미지 라이선스, 데이터 출처 표기 등)를 점검하고 수정하는 단계까지 나아갔습니다. 기획-개발-배포로 끝나는 게 아니라 **운영 관점의 문제까지 스스로 찾아 해결**해야 한다는 걸 배운 프로젝트였습니다.

### 5. 수익성은?

현재는 수익 모델이 없는 무료 서비스입니다. 결제 없이 배포 중이며, 제휴 마케팅·배너 광고·프리미엄 기능·B2B 리드·하드웨어 업체 제휴·API 라이선싱 등 6가지 후보를 검토했고, 지금 트래픽 규모에서는 **제휴 마케팅**부터 시도하는 게 현실적이라고 판단했습니다. 자세한 비용 구조·모델별 장단점·단계별 실행 계획은 [수익성 검토](#수익성-검토) 섹션에 정리했습니다.

## 주요 기능

- **8개 카테고리 순차 선택**: CPU → 메인보드 → RAM → SSD → GPU → PSU → 케이스 → 쿨러
- **실시간 호환성 검증**: 소켓, 폼팩터, RAM 규격/속도, GPU 길이, PSU 전력·폼팩터·커넥터 세대(12VHPWR/12V-2x6), 쿨러 지원 소켓·높이·라디에이터 크기, M.2/PCIe 버전까지 검사
- **성능 병목 진단**: CPU-GPU 등급 격차를 계산해 병목 가능성을 경고
- **실시간 가격 조회**: 다나와 최저가를 크롤링해 부품별/합계 가격 표시 (`Crawl-delay: 10` 준수, 24시간 캐시)
- **AI 사전 추천 마법사**: Groq(Llama 3.3 70B)에게 조합을 요청 → 위험 조합 자체 검증 → 재시도 → 실패 시 안전 조합 폴백, 전체 60초 예산 내 처리
- **제품 이미지 연동**: 네이버쇼핑 검색 API로 부품 실사 이미지 표시
- **미쿠 에디션 이스터에그**: 선택 가능한 모든 카테고리를 하츠네 미쿠 에디션으로만 채우면 히든 업적 발동
- **개인정보처리방침 / 이용약관 페이지**, 광고 슬롯 포함

## 호환성 검증 규칙

| 항목 | 규칙 |
|---|---|
| CPU ↔ 메인보드 | 소켓 일치 |
| 메인보드 ↔ RAM | 규격(DDR4/5), 최대 속도, 최대 용량 이내 |
| GPU ↔ 케이스 | GPU 길이 ≤ 케이스 최대 GPU 장착 길이 |
| GPU ↔ PSU | PSU 용량 ≥ GPU 권장 용량, 커넥터 세대(12VHPWR/12V-2x6) 호환 |
| PSU ↔ 케이스 | PSU 폼팩터 일치 |
| 쿨러 ↔ CPU | 소켓 지원 여부, 최대 TDP 지원량 |
| 공랭 쿨러 ↔ 케이스 | 쿨러 높이 ≤ 케이스 최대 쿨러 높이 |
| 수랭 쿨러 ↔ 케이스 | 라디에이터 크기가 케이스 지원 목록에 포함 |
| SSD ↔ 메인보드 | NVMe인 경우 M.2 슬롯 존재, PCIe 버전 이내 |
| 메인보드 ↔ 케이스 | 폼팩터가 케이스 지원 목록에 포함 |

경고는 3단계(초록/노랑/빨강)로 표시되며, 빨강이어도 선택 자체를 막지 않습니다.

## 데이터 출처 및 라이선스

- **부품 스펙·가격**: 다나와 (검색 결과 최초 1건 기준, 크롤링 정책 준수)
- **제품 이미지**: 네이버쇼핑 검색 API, 각 제조사
- **한글 폰트**: 마루 미냐 한글 (OFL 라이선스 — 상업적 이용·임베드 가능, 출처 표기 불필요, 수정·재배포 가능(OFL-1.1 채택 조건), 단독 판매만 금지)

- **광고 배너 이미지**: 원저작자에게 사용 허가받음
- **AI 추천**: Groq API (Llama 3.3 70B) 호출, 응답은 참고용 추천이며 최종 호환성은 자체 검증 로직을 통과한 것만 노출

## 수익성 검토

> 아래는 확정된 수익 모델이 아닌 **검토용 제안 및 분석 자료**입니다. TriFit은 현재 내일배움캠프 PM 부트캠프 과제로 개발·배포된 무료 개인 프로젝트이며, 사용자에게 어떠한 결제도 요구하지 않습니다.

### 현재 비용 구조 (수익 없이 지출만 있는 구조)

- Vercel 호스팅 — 무료 티어
- Supabase(PostgreSQL DB) — 무료 티어
- AI 추천 API(Groq) — 무료 티어
- 네이버쇼핑 API(제품 이미지) — 무료 사용 범위 내
- 1인 개발 인건비 — 별도 지출 없는 기회비용

트래픽이 늘면 각 서비스의 무료 티어 사용량을 초과해 유료 전환이 필요할 수 있습니다.

### 수익화 관점에서 본 보유 자산

- **트래픽·방문자**: PC 조립을 준비 중인, 구매 의도가 뚜렷한 사용자층
- **자체 구축 부품 DB**: 8개 테이블, 다나와·제조사 출처의 검증된 스펙 데이터
- **호환성 판정 로직**: 규칙 기반 호환성 체크 엔진
- **AI 추천 기능**: 사용자 예산·용도 기반 부품 추천 결과

구매 의도가 뚜렷한 트래픽은 커머스 연계형 수익 모델(제휴 커미션 등)에 특히 유리합니다.

### 손익 시뮬레이션 템플릿

아직 실제 트래픽·전환 데이터가 없어 구체적인 수익 금액은 추정하지 않았습니다. 데이터 확보 후 아래 구조에 값을 채워 산출하는 것을 제안합니다.

| 항목 | 값 | 비고 |
|---|---|---|
| 월간 방문자 수(MAU) | 직접 입력 필요 | 실제 트래픽 데이터 확보 후 기입 |
| 구매 전환율(%) | 직접 입력 필요 | 제휴 링크 클릭률·구매 전환 데이터 필요 |
| 건당 평균 커미션 단가 | 직접 입력 필요 | 제휴사 정책에 따라 상이(구매액의 1~5% 수준이 일반적) |
| 월 예상 수익 | = MAU × 전환율 × 건당 단가 | 위 세 값 입력 시 자동 산출 가능 |
| 월 예상 비용 | 직접 입력 필요 | 무료 티어 초과분 발생 시 기입 |
| 월 순이익(추정) | = 예상 수익 − 예상 비용 | - |
